# Phase 3A — Design Partner Readiness (durable decisions)

> Review-only. The bar here is winning **3–5 hand-held design partners**, not scale. At this stage the founder controls the demo environment, so most enterprise gaps are *framing*, not blockers.

## Verdict
**GO — conditional on demo preparation.** The core works, the governance story is credible, and the magic moment lands in < 5 minutes.

## Durable decisions
- **ICP for partners:** WordPress **agencies/operators running AI on client sites**, and **AI-forward teams wiring MCP agents**. **Do NOT recruit hobby/solo freelancers** (governance is a vitamin for them; their feedback mis-steers).
- **The demo is the product:** AI proposes → review the diff → approve → applies → **one-click undo** in History, fully audited. Plus the Door-2 beat: a client/agent action **stops at Approvals**.
- **Hide at demo time:** raw/engine internals, Patches (heavy), the dense Tokens matrix, any "can't generate yet" provider, the enable/key setup (pre-seed it).
- **Pre-seed + rehearse:** a real Anthropic key, one enabled tool, a sandbox post + image, a reset path, and an API-hiccup fallback. *(Most demo failures are operational, not product.)*
- **Set expectations honestly up front:** Anthropic-only generation, single-site, no SSO yet, keys-at-rest.

## Assets defined (reusable)
- **Demo checklist**, **10-minute demo script** (fear → relief → trust → power → desire), **pilot success criteria** (real governed change on a real site + undo used + a door validated + unprompted return), and **feedback questions** (which door matters; what's the one thing to put it on a real client site; willingness to pay).

## Carried forward
- Phase 4 then removed the last technical blocker (in-admin enablement), making the hand-held demo runnable end-to-end without code editing.
